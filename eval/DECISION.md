# 로컬 에이전트 선정 — 결정 (2026-07)

**결정: `Qwen2.5-VL-7B` + 헤더인식 프롬프트** (Ollama / Jetson AGX Orin 32GB)
구조: 비전추출 → 계좌합계 대조 게이트 → 결정적 엔리치(주가=평가금액/수량, 계좌 간 수량 역산) → 검색식별(증권사·티커).

## 왜 7B (요지)
1. 헤더 프롬프트 적용 시 **조밀·클린 레이아웃 모두 평가금액·재현율 100%**.
2. **32B 불필요** — 상한(100%) 초과 불가 + 메모리 3배·속도 저하 (조밀 표에선 3B>7B로 "크기=정확도" 반증).
3. **3B 아님** — 값은 정확하나 종목 누락(재현율 8/9); 32GB Orin엔 메모리 아낄 이유 없음.
   → 정확도·재현율 100%를 **가장 가볍게 만족하는 지점 = 7B**.

## 측정 스코어 (실제 스크린샷 8장, 앱 실 프롬프트, 하드코딩 없음)
| 모델·프롬프트 | 클린 평가금액 | 조밀 평가금액 | 조밀 매수금액 | 재현율 |
|---|---|---|---|---|
| 7B 원본 | 100% | **11%** ❌(열 오매핑) | 11% | 100% |
| 3B 원본 | 100% | 100% | 100% | **92%(1누락)** |
| **7B + 헤더** ★ | 100% | **100%** | 100%\* | **100%** |

\*매수금액 12/13 — 1건은 현금행 `0 vs null`(실질 오류 아님).

## 검토 모델과 단점
- **Qwen2.5-VL-7B(원본)** — 클린 완벽, 조밀 표 열 오매핑 → 헤더 프롬프트로 교정 후 채택.
- **Qwen2.5-VL-3B** — 값 정확하나 종목 1개 누락(재현율).
- **Qwen2.5-VL-32B** — 16GB 평가기 OOM으로 미실행; 7B가 이미 100%라 필요 없음.
- **Llama 3.2 Vision / MiniCPM-V / Gemma 3 / Granite-Vision** — 후보였으나 7B가 목표 충족해 미실행(추가 실익 없음).

## 핵심 교훈
- **크기 ≠ 정확도** (실측으로 3B>7B on 조밀 레이아웃).
- **일반 프롬프트(헤더 읽기)만으로 11%→100%** — 증권사별 하드코딩 불필요.
- **합계검증 게이트**로 비전 비결정 편차를 런타임 방어.

## 남은 검증
Orin 실기에서 ①속도 ②요약/현금 화면 환각저항 ③반복 재현성 확인 후 배포.
