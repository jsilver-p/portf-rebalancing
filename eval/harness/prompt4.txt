증권사 앱 스크린샷에서 보유자산 표를 추출하라. 먼저 증권사와 화면 유형(일반/연금저축/IRP/퇴직연금/ISA)을 파악한 뒤, 각 보유 행을 JSON 배열로만 출력하라. 다른 텍스트·코드펜스 금지.
스키마: [{"broker":<화면에 보이는 증권사·브랜드·계좌 라벨 그대로, 없으면 null>,"accountType":<일반|연금저축|IRP|퇴직연금|ISA, 없으면 null>,"name":<개별 종목·항목 이름>,"assetClass":<성장주|배당주|채권|금·원자재|현금|부동산|혼합(주식·채권)>,"currency":<KRW|USD>,"qty":<수량 또는 null>,"price":<현재가 또는 null>,"value":<평가금액 또는 null>,"cost":<매수금액 또는 null>,"pnl":<평가손익 또는 null>,"confidence":<0~1>}]
<> 안은 그 칸에 무엇을 넣으라는 설명이다. 설명 문구 자체를 값으로 출력하지 마라 — 화면에서 읽지 못한 값은 반드시 null이다.
표 읽는 법(중요): 값을 채우기 전에 먼저 열 제목(헤더) 행을 읽어 각 열이 무엇인지 파악하라. 각 숫자를 헤더가 가리키는 항목(평가금액/매수금액/평가손익/수익률/수량/현재가)에 정확히 매핑하라. 한 칸(셀)에 숫자가 위아래로 여러 개 쌓여 있으면 그 열 헤더에 적힌 순서(예: '평가금액/매수금액' 헤더면 위=평가금액, 아래=매수금액)대로 각각 배정하라. 평가손익(수익/손실 증감액)이나 수익률(%)을 평가금액·매수금액·수량으로 착각하지 마라. value에는 반드시 평가금액을, cost에는 반드시 매수금액(매입원금)을, qty에는 반드시 수량(주)만 넣어라.
헤더가 없는 목록형 화면(상품별 총액·계좌별 잔고)에서 금액이 위아래로 쌓여 있으면 **위=평가금액(잔고), 아래=평가손익**이다. value에 손익을 넣지 마라.
매수금액 열이 아예 없고 평가손익만 있는 화면이면 cost=null로 두고 손익은 pnl에 넣어라(cost를 손익으로 채우지 마라).
숫자 표기: 순수 JSON 숫자만 — 쉼표·통화기호(원/$)·%·"주" 같은 단위와 따옴표를 넣지 마라. 양수에 + 부호를 붙이지 마라(+263504는 잘못, 263504가 맞다). 음수만 -를 쓴다. 값이 없으면 JSON null.
규칙: (1) 화면에 없는 값은 null — 절대 추측 금지. (2) 합계·헤더 행 제외. 단 화면이 '상품별 총액'이나 '계좌별 잔고' 목록이면 그 항목들을 한 행씩 출력하라(그 화면엔 개별 종목이 없다). (3) name에는 항목 이름만 — 계좌번호·예금주(사람 이름)·금액을 이름으로 쓰지 마라. 예수금·현금 화면은 화면에 적힌 항목명(예: 원화예수금·외화예수금)을 쓴다. (4) 같은 자산을 단위만 바꿔 두 번 출력하지 마라(외화 잔액을 달러 행과 원화 행으로 중복 금지) — 한 자산 = 한 행. (5) 수량 없이 매입원금/평가금액만 있는 화면(연금 등)은 value·cost만 채우고 qty=null. (6) assetClass는 상품명으로 판단: 나스닥/S&P/성장/테크/AI→성장주, 고배당/배당→배당주, 채권→채권, 금현물/원자재→금·원자재, 현금성/예수금/CMA→현금, 리츠→부동산, 채권혼합→혼합(주식·채권). (7) 값이 흐릿하거나 잘렸으면 해당 행 confidence를 0.7 이하로.
